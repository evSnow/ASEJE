Lesson One: Activity Bar Overview

In this section we will explain the basics of the Activity Bar and the function of each icon. This is important because these icons help complete important tasks and improve workflow. A more detailed explanation will be given in another lesson.

What is it?

The Activity Bar is a toolbar on either the left, right, or top of the VS Code window. Each icon has a function that developers frequently use when coding.

Explorer

This function allows users to open a folder and manage the contents of a directory. It includes adding new files, renaming folders, and navigating the project.

Source Code

This icon controls Git versioning for GitHub, enabling quick push/pull actions and staging changes.

Run and Debug

This function is used to debug code, create breakpoints, and run the program. Debugging can also be enhanced by other extensions.

Extensions

This opens the marketplace where users can install extensions created by others. Developers can publish their own extensions with documentation and GitHub links.

Understanding the Action Bar (Sidebar)

The Action Bar (also known as the Activity Bar) is the vertical toolbar on the left side of Visual Studio Code. It provides access to the most important tools used by developers. Understanding this area is essential for navigating VS Code.

What the Action Bar Contains

  • Explorer – view and manage project files
  • Search – search text across the entire project
  • Source Control – interact with Git (commit, push, pull)
  • Run and Debug – start and control debugging sessions
  • Extensions – install and manage additional VS Code features
  • Custom ASEJE Views – added by our extension to support beginners

How ASEJE Uses the Sidebar

  • Highlighting breakpoints clearly
  • Showing the current execution line
  • Providing simplified debug controls
  • Helping beginners step through code in a guided way

Debugging Controls in the Action Bar

  • Start / Continue – run or resume the program
  • Pause – temporarily halt execution
  • Stop – end the debug session
  • Step Over – go to the next line without entering a function
  • Step Into – go inside a function call
  • Step Out – exit the current function

Example: Debugging Using the Sidebar

  1. Open the file you want to debug.
  2. Click the Run and Debug icon in the Action Bar.
  3. Click next to a line number to set a breakpoint.
  4. Press Run and Debug to begin.
  5. Use Step Over / Step Into / Step Out to control execution.
