# Contributing Guidelines

Thank you for contributing to the ASEJE extension project!

## Code Style
- Follow consistent JavaScript formatting.
- Use meaningful variable and function names.
- Add comments to functions when needed for clarity.

## Git Commit Rules
- Keep commit messages short and descriptive.
- Use present tense verbs (e.g., *Add*, *Fix*, *Update*).

## Branching
- Most small fixes can be pushed to `main`.
- For larger features, create a separate branch.

## Reporting Issues
- Use clear descriptions.
- Include steps to reproduce the issue if possible.

These guidelines help maintain a clean and consistent codebase.
